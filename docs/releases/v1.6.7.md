# v1.6.7 — Supermemory Memory API (Opt-in)

**Release date:** 2026-06-10

Supermemory integration for memory persistence across AI coding sessions.

---

## What's New

### Supermemory Memory API (Opt-in)

This release introduces optional integration with Supermemory, a CLI tool that
provides persistent memory across AI coding sessions. The memory persists session
context, decisions, and learnings so the agent never loses track.

**Key features:**
- `opk supermemory install` — install @supermemory/ai via npm global install
- `opk supermemory status` — check current status
- `opk supermemory init` — init Supermemory in the current project
- `/supermemory-init` — agent command for initialization workflow

## Upgrade Path

Users with an existing install can update via:

```bash
opk up          # Update kit + project
```

Or manually:

```bash
cd ~/opencode-power-kit
git pull --ff-only
bash install-global.sh --yes   # Update global config
```

No breaking changes. Supermemory is optional and opt-in only.

## Safety

- Supermemory is never auto-installed (not during `opk up`, bootstrap, or
  shell startup)
- Install scripts never use sudo, never curl|sh, never vendor source
- `opk supermemory install` requires user confirmation (or `--yes`)
- Agent command `/supermemory-init` always routes through `opk supermemory`
  wrapper — agents never self-install npm packages
- `opk supermemory status` is read-only
