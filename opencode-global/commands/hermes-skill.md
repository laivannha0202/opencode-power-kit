# /hermes-skill

Structured skill improvement practice.

## When To Use

- Khi muốn practice structured thinking
- Khi cần improve code quality patterns
- Khi muốn review và improve prompts/docs/error messages
- Khi cần reflection practice

## Focus Areas

### code-quality
Review recent code changes và identify:
- Code smells
- Anti-patterns
- Type safety gaps
- Error handling gaps
- Performance issues
- Accessibility issues

### writing
Review và improve:
- Agent prompts (rõ ràng, cụ thể, testable)
- Error messages (actionable, informative)
- Documentation (đúng audience, clear)
- Commit messages (conventional, descriptive)

### reflection
Structured reflection exercise:
- Pre/Post task comparison
- Assumption audit
- Tool choice analysis
- Time management review

## Workflow

```markdown
1. Chọn focus area
2. Collect sample (recent code/docs/prompts)
3. Analyze against best practices
4. Propose specific improvements
5. Apply improvements (với user approval)
6. Verify improvements work
```
